Comic-Web

Comic-web is a markup language for describing comics, as well as a rendering engine for this language.

The markup language

The Comic-Web markup language is quite simple, you can understand it in minutes.

Layers

Every word that is capitalized and/or followed by an empty line describes a layer in the panel.

The visualizer searches an image that goes by the name of the layer in the designited folder, only the file names are lowercase and use dashes instead of spaces as separators, (so 'THE STUDIO', becomes 'the-studio.svg'.

Stacking

Each successive layer goes on top of the last one.

You can stack as many layers as you want, the only thing is that they should be transparent, else they would just overwrite each other.

Characters

When you have a layer line, followed by some other lines that are not set in all-caps (and have no empty line between them), the engine assumes that the layer is a character, and the lines are... their lines.

If so, the image (named "jencel.svg" in this case) is scalled a bit, so the text panel is visible.

Moods

To help you organize your files better, the engine supports setting "moods" to characters and other images, by enclosing a mood in brackets, next to the character's name. The location where the engine will search for is "/", so, for example the image below is at "./jencel/sad.svg". If the image doesn't exit, it would render "./jencel.svg".

The moods can also be a comma-separated list, where the logic is the same: "JENCEL (sad, closeup)" would render "./jencel/sad/closeup.svg". If it doesn't find it, it would render "./jencel/sad.svg" and (if that also doesn't exist, just "./jencel.svg").

You can understand it better if you check how the folder of this comic is organized

Stacking characters

You can have more than one characters, the engine would do its best to place them and their speech balloons properly.

When there are more than one character, the engine places the characters and the speech balloons next to each other, according to the way they are defined.

Stacking characters CONTD

You can have many characters on-screen and there isn't any special way to describe characters, any image can be a character.

The way that the characters are arranged corresponds to their lines.

Narrators and off-screen dialogue

There is a special 'NARRATOR' character who is always off screen.

And, if you want to have some of the regular characters talking off-screen, there is also a special "off-screen" mood which won't render anything.

The narrator's lines are always rendered at the top and bottom of the panel, as is customary for comics.

Custom images

If you don't want to use the rendering engine for some panel, you can always paint a whole complete panel and include it.

And if you want to draw a custom panel, but still use the speech balloons, you can make use of the "off-screen" mood.

The toolchain

So, you know about the markup, not let's find out how can we use it.

The API

The API of Comic Web is exported by lib/index.js and currently it consists of a single JavaScript function, called renderComic accepting a string containing a description of the panel in the format that we described and a directory, where the images are stored. The function returns an SVG DOM element, with the panel.

The plugin

The plugin is the principal way of using the markup. Basically it consists of the following steps:

Include the plugin file in your HTML page.

In the file, insert an image, leading to a non-existent link in your images dir, and put the script in the alt-text property.

The plugin would generate a panel in the place of the image.

Styling

Besides the images, the engine also embeds a style.css file into all images. This file is used for styling the frames, balloons etc.
